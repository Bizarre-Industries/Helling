# Contributing

Thanks for your interest in contributing to Helling.

## Ground Rules

- Follow docs-first behavior: specs and ADRs are authoritative.
- Keep changes minimal and focused.
- Use conventional commit messages.
- Include rationale for scope changes.

## Workflow

1. Fork and branch from main.
2. Make focused changes.
3. Update docs and ADR references when behavior changes.
4. Open a pull request with clear scope and validation notes.

## Commit Format

Use conventional commits:

- `feat:`
- `fix:`
- `docs:`
- `refactor:`
- `test:`
- `chore:`

## Pull Request Checklist

- Scope is explicit
- Docs updated if behavior changed
- No unrelated refactors
- Security-impacting changes called out

# Security Policy

## Reporting a Vulnerability

Please report vulnerabilities privately by email to the maintainers listed in the repository ownership records.

Include:

- Affected component
- Reproduction steps
- Impact assessment
- Suggested mitigation if available

## Disclosure Process

- Initial triage acknowledgement target: 72 hours
- Fix timeline depends on severity and exploitability
- Coordinated disclosure is preferred

## Scope

In scope:

- Authentication and authorization flaws
- Privilege escalation paths
- Sensitive data exposure
- Supply-chain integrity issues in release or CI artifacts

Out of scope:

- Social engineering attacks
- Vulnerabilities in unsupported forks

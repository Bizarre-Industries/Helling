## TLDR / 30 seconds

v0.1 said _"the WebUI looks like Proxmox crossed with a tactics console — and that works."_ Still true. The bigger risk, missed in v0.1, is that the implementation walked away from the spec while the spec kept being treated as truth. `docs/design/philosophy.md` + `docs/design/tokens.md` + `docs/spec/webui-spec.md` all specify React 19 + antd 6 + pro-components. The code is hand-rolled inline-styled JSX with no antd anywhere. `docs/spec/auth.md` §2.2 says access tokens live in memory only. `auth-store.ts` puts them in localStorage. `docs/spec/accessibility.md` commits to WCAG 2.1 AA. The implementation has zero ARIA attributes.

### If you do nothing else from this report

1. **Decide on the stack before anything else.** Resolved Phase 0 by ADR-051 → spec.
2. **Fix auth-store before merging another PR.** Resolved Phase 1 (F-37).
3. **Add one error boundary at root.** Resolved Phase 1 (F-39).
4. **Decide on the data layer before adding more pages.** Phase 3 (F-01).
5. **Split `pages.jsx` + `pages2.jsx`.** Phase 2 in flight (F-05).
6. **Add an a11y pass before more polish.** Phase 5 (F-25/F-26/F-27/F-28).

## Section 00 — Reframings: where v0.1 points at the wrong fix

### R-01 → F-03 — SSE is the right answer, not `refetchInterval`

**Refs:** `api/openapi.yaml` operationId: `eventsSse`

v0.1 prescribes TanStack's `refetchInterval` for the 1.5s mock loop. Polling 6+ resource types every 1.5s is exactly the load profile F-03 was trying to escape. Build a `useEventsStream()` hook that opens an EventSource on `/api/v1/events`, dispatches by event type, and updates the relevant query keys. `refetchInterval` stays as the fallback for SSE-blocked clients.

**Effort:** M · 2 days

### R-02 → F-06, F-14, F-23, F-25, F-26 — primitive-extraction findings assume the current stack stays

**Subsumed by:** F-36 (Stack drift)

Park behind the F-36 decision. ADR-051 chose spec, so these collapse into "port pages onto pro-components" in Phase 4.

### R-03 → F-22 — viewport-meta is actively wrong, not just restrictive

**Refs:** `index.html` L4

`<meta name="viewport" content="width=1440">` is non-standard. Breaks pinch-zoom on touch. **Fix shipped Phase 1:** `width=device-width, initial-scale=1` + CSS gate at 1440px.

**Effort:** XS · 30 min · ✅ shipped

### R-04 → F-29 — bundles two unrelated wins, one needs verification

Code-splitting is a clean ~1 day win. Unbounded font: verify it's actually downloaded before claiming the savings. Subsetting Big Shoulders Stencil to Latin-1 + digits is real.

**Effort:** S–M · 1 day net

### R-05 → F-04 — right but undersells how broken auth feedback is

Sequence F-38 (wire login to `authLogin`) before F-04's error-state pass. **F-38 closed Phase 1; F-04 unblocked.**

## Section 02 — Data layer & API integration

There's a hey-api fetch client wired up (`web/src/api/client.ts`), TanStack QueryProvider mounted at root, an auth store, and exactly two query hooks hitting real endpoints (`useInstancesQuery`, `useContainersQuery`). Both feed exactly one consumer: `useDashboardCounts`. **Every other page reads from and writes to in-memory mock arrays declared in `shell.jsx`.**

> **Recommendation.** Replace the mock arrays with a typed mock _adapter_ behind the same `useQuery` shape that `queries.ts` already uses.

### F-01 — components own mock state directly 🔴 BLOCKER

**Files:** `shell.jsx` · `pages.jsx` · `pages2.jsx`
**Refs:** `shell.jsx` L84–L168, `queries.ts` L65–L87

Page components import `INSTANCES`, `CONTAINERS`, etc. directly. `useStore()` is a global re-render trigger. No hook boundary to swap behind.

**Fix:** Stand up `useInstancesQuery`, `useInstanceMutation`, `useTasksQuery`, `useAlertsQuery`. Back with MSW or dev-only handler in `client.ts`.

**Effort:** M-L · ~1 sprint · **Impact:** ●●●●●

### F-02 — mock data shape doesn't match real API shape 🔴 BLOCKER

**Refs:** `shell.jsx` INSTANCES; `queries.ts` IncusInstance

Incus returns `status: "Running"` (PascalCase). Mocks use lowercase. Dashboard hook silently filters on `'Running'`; every other page on `'running'`.

**Fix:** Define one canonical `Instance` type and a normaliser at the API boundary.

**Effort:** S · 1–2 days · **Impact:** ●●●●○

### F-03 — 1.5s mock tick mutates global state 🟠 MAJOR

**Refs:** `shell.jsx` L171–L208 · v0.2 see R-01

`setInterval` at module load drifts every running instance every 1.5s, regardless of which page is visible.

**Fix:** SSE consumer keyed off `eventsSse`. Polling is fallback.

**Effort:** M · 2 days · **Impact:** ●●●○○

### F-04 — errors and loading states are decorative 🟠 MAJOR

**v0.2 see R-05** — sequenced after F-38 (✅ Phase 1).

Dashboard treats query errors as "fall back to mocks." Other pages have no loading skeletons, no error states.

**Fix:** Three explicit list-page states (loading skeleton, empty, error). `infra.jsx EmptyState` already there — needs an "error" variant.

**Effort:** M · 2–3 days · **Impact:** ●●●●○

### F-42 — OpenAPI ships `eventsSse` but no SSE consumer 🟠 MAJOR · NEW

**Refs:** `api/openapi.yaml` operationId list

49 operationIds; `eventsSse` is one. WebUI grep for `EventSource`: 0 hits. **Backend currently snapshot-GET — full streaming v0.1-beta.** Plan ships in two stages: snapshot-poll first, EventSource upgrade when backend ready.

**Fix:** `useEventsStream()` opens EventSource, dispatches by event type, updates matching query keys. Reconnect with exponential backoff.

**Effort:** M · 2 days · **Impact:** ●●●●○

### F-43 — mock fidelity gap is shape-level 🟠 MAJOR · NEW

Real Incus instance objects carry `architecture`, `config`, `created_at`, `description`, `devices`, `ephemeral`, `expanded_config`, `expanded_devices`, `last_used_at`, `location`, `profiles`, `project`, `stateful`, plus a nested `state` substructure. Current mock is flat ~8-field.

**Fix:** Generate `Instance` type from Incus OpenAPI (or hand-write from official docs). Seed mocks from real shape.

**Effort:** S · 1 day per resource type · **Impact:** ●●●●○
